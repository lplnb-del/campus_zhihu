## 一、项目总体概述

### 1.1 项目背景与意义

随着高校信息化建设的深入推进，校园内的知识共享与学术交流需求日益增长。现有的通用社交平台（如知乎、贴吧）无法完全满足高校场景下的专业需求，存在内容质量参差不齐、专业匹配度低、积分激励机制缺失等问题。

本项目**CampusZhihu（校园知乎）**应运而生，致力于打造一个面向高校师生的**垂直领域知识问答社区**。通过实名认证、专业标签分类、积分悬赏机制等特色功能，构建高质量的学术交流平台。

### 1.2 项目目标

| 目标维度 | 具体内容 |
|---------|---------|
| **功能目标** | 实现问题发布、回答互动、二级评论、标签分类、积分悬赏等核心功能 |
| **技术目标** | 掌握Spring Boot + Vue 3前后端分离架构，实践RESTful API设计 |
| **学习目标** | 深入理解JWT认证、MyBatis-Plus ORM、Redis缓存等技术栈 |

### 1.3 技术选型

本项目采用**前后端分离架构**，技术栈选择充分考虑了课程学习要求和项目实际需求：

**后端技术栈**：
- Spring Boot 3.2.0：简化Spring应用开发，提供自动配置
- MyBatis-Plus 3.5.5：简化数据库操作，提供代码生成、分页等功能
- Spring Security + JWT：实现无状态的身份认证
- MySQL 8.0：关系型数据库，支持事务和外键约束
- Redis：缓存热点数据，实现接口限流

**前端技术栈**：
- Vue 3.4.0：采用Composition API，提升代码复用性
- Element Plus：企业级UI组件库，保证界面美观度
- TypeScript：提供类型安全，减少运行时错误
- Vite：新一代构建工具，提升开发体验

### 1.4 项目特色

1. **双Token认证机制**：Access Token（15分钟）+ Refresh Token（7天），兼顾安全性与用户体验
2. **积分悬赏系统**：提问者可悬赏积分激励优质回答，形成正向激励循环
3. **二级评论支持**：支持对评论进行回复，实现深度讨论
4. **智能标签推荐**：基于问题内容自动匹配标签，提升内容可发现性

## 四、其他（技术亮点与难点）

### 4.1 双Token认证机制

**问题**：传统的Session认证无法适应前后端分离架构，单一Token难以平衡安全性与用户体验。

**解决方案**：采用Access Token + Refresh Token双Token机制
- Access Token：有效期15分钟，用于API访问，存储在内存
- Refresh Token：有效期7天，用于刷新Access Token，存储在Redis

```java
// JwtUtil.java 核心逻辑
public String generateAccessToken(Long userId) {
    return Jwts.builder()
        .setSubject(userId.toString())
        .setExpiration(new Date(System.currentTimeMillis() + 15 * 60 * 1000))
        .signWith(key)
        .compact();
}
```

### 4.2 接口限流保护

**问题**：防止恶意用户频繁调用接口，保护系统稳定性。

**解决方案**：基于Redis + AOP实现接口限流

```java
@RateLimiter(time = 60, count = 10) // 60秒内最多10次请求
public Result<UserVO> getUserInfo() {
    // 业务逻辑
}
```

### 4.3 二级评论实现

**问题**：评论的回复（楼中楼）如何高效存储和查询？

**解决方案**：采用parent_id字段存储父评论ID，通过递归查询实现树形结构

```java
// 递归构建评论树
private List<CommentVO> buildCommentTree(List<CommentVO> comments) {
    Map<Long, List<CommentVO>> parentMap = comments.stream()
        .filter(c -> c.getParentId() != null)
        .collect(Collectors.groupingBy(CommentVO::getParentId));
    // 递归组装...
}
```

### 4.4 积分事务保证

**问题**：采纳回答时需要同时扣除提问者积分、增加回答者积分，如何保证一致性？

**解决方案**：使用Spring @Transactional注解保证事务一致性

// UserController.java - login方法
@PostMapping("/login")
public Result<LoginVO> login(@RequestBody LoginDTO dto) {
    // 1. 根据邮箱查询用户
    SysUser user = userService.getByEmail(dto.getEmail());
    // 2. 验证密码（BCrypt）
    if (!passwordEncoder.matches(dto.getPassword(), user.getPassword())) {
        return Result.error("密码错误");
    }
    // 3. 生成双Token
    String accessToken = jwtUtil.generateAccessToken(user.getId());
    String refreshToken = jwtUtil.generateRefreshToken(user.getId());
    // 4. 存储RefreshToken到Redis（7天过期）
    redisUtil.set("refresh_token:" + user.getId(), refreshToken, 7 * 24 * 60 * 60);
    // 5. 返回给前端
    return Result.success(new LoginVO(accessToken, refreshToken, user));
}

// QuestionController.java - publish方法
@PostMapping("/publish")
public Result<Long> publish(@RequestBody QuestionPublishDTO dto) {
    // 1. 从JWT中提取当前用户ID
    Long userId = UserContext.getCurrentUserId();
    // 2. 创建问题实体
    BizQuestion question = new BizQuestion();
    question.setUserId(userId);
    question.setTitle(dto.getTitle());
    question.setContent(dto.getContent());
    question.setImages(dto.getImages());
    question.setRewardPoints(dto.getRewardPoints());
    // 3. 保存问题（MyBatis-Plus自动填充创建时间）
    questionService.save(question);
    // 4. 保存标签关联关系
    if (dto.getTagIds() != null) {
        dto.getTagIds().forEach(tagId -> {
            QuestionTag questionTag = new QuestionTag(question.getId(), tagId);
            questionTagService.save(questionTag);
        });
    }
    // 5. 扣除悬赏积分（事务保证）
    if (dto.getRewardPoints() > 0) {
        userService.deductPoints(userId, dto.getRewardPoints());
    }
    return Result.success(question.getId());
}

// QuestionServiceImpl.java - acceptAnswer方法
@Transactional(rollbackFor = Exception.class)
public void acceptAnswer(Long questionId, Long answerId) {
    // 1. 查询问题和回答
    BizQuestion question = getById(questionId);
    BizAnswer answer = answerService.getById(answerId);
    // 2. 验证权限（只有提问者可采纳）
    if (!question.getUserId().equals(UserContext.getCurrentUserId())) {
        throw new BusinessException("无权限操作");
    }
    // 3. 标记回答为已采纳
    answer.setIsAccepted(1);
    answerService.updateById(answer);
    // 4. 更新问题状态为已解决
    question.setIsSolved(1);
    updateById(question);
    // 5. 转移悬赏积分（提问者→回答者）
    if (question.getRewardPoints() > 0) {
        userService.transferPoints(
            question.getUserId(),    // 转出
            answer.getUserId(),      // 转入
            question.getRewardPoints()
        );
    }
    // 6. 发送系统通知给回答者
    noticeService.sendAcceptNotice(answer.getUserId(), questionId);
}

// request.ts - Axios响应拦截器
axios.interceptors.response.use(
  response => response,
  async error => {
    const originalRequest = error.config;
    // 401错误且未重试过
    if (error.response?.status === 401 && !originalRequest._retry) {
      originalRequest._retry = true;
      // 调用刷新Token接口
      const refreshToken = localStorage.getItem('refresh_token');
      const { data } = await axios.post('/api/user/refresh-token', { refreshToken });
      // 存储新Token
      localStorage.setItem('access_token', data.data.accessToken);
      // 重试原请求
      return axios(originalRequest);
    }
    return Promise.reject(error);
  }
);
