### 实验分工情况

**1、分工角色**：后端核心模块开发负责人

**2、工作内容描述**（按实验过程描述，200字以内）

负责系统后端核心模块的开发工作，包括用户认证模块（UserController）、问题管理模块（QuestionController）和首页聚合模块（IndexController）。实现了基于JWT的双Token认证机制，完成用户注册、登录、Token刷新等核心功能；开发了问题发布、列表查询、搜索、采纳回答等接口，使用MyBatis-Plus实现分页查询；实现了Spring Security安全配置和JwtFilter过滤器，确保接口访问安全。同时负责全局异常处理器和统一响应结果封装，编写了对应的Junit单元测试用例。累计完成Controller层代码约1500行，Service层代码约800行，编写测试用例30余个。

### 个人实验总结（归纳经验教训，总结不足与可能改进之处）

**经验收获**：
1. 通过本次项目深入理解了Spring Boot的自动配置原理，掌握了我Spring Security与JWT的整合使用，理解了无状态认证的优势。
2. 熟练掌握了MyBatis-Plus的使用，包括分页插件、代码生成器、条件构造器等功能，大大提升了开发效率。
3. 学习了Redis在项目中的实际应用，包括缓存热点数据、存储RefreshToken、实现接口限流等场景。
4. 理解了前后端分离架构的优势，通过RESTful API设计，前端和后端可以独立开发和部署。

**存在不足**：
1. 初期对JWT Token的过期处理考虑不够周全，导致用户在使用过程中频繁掉线，后来通过双Token机制和Axios拦截器实现自动刷新才解决。
2. 数据库查询优化不足，问题详情页存在N+1查询问题，导致响应时间较长，后续需要通过联表查询或缓存优化。
3. 单元测试覆盖率不够，部分业务逻辑没有编写测试用例，导致上线后出现了一些边界条件下的bug。

**改进方向**：
1. 深入学习数据库优化技巧，使用EXPLAIN分析慢查询，合理使用索引和联表查询。
2. 提高单元测试覆盖率，使用Mock框架模拟依赖，确保核心业务逻辑的可靠性。
3. 学习使用Postman等工具进行接口测试，确保API的健壮性和文档完整性。
4. 加强代码审查，使用SonarQube等工具进行静态代码分析，及时发现潜在问题。

2. 学会了使用ER建模工具进行数据库设计，理解了实体间的一对一、一对多、多对多关系。
3. 掌握了MyBatis的复杂查询，包括联表查询、子查询、动态SQL等，实现了评论树形结构的递归查询。
4. 学习了MinIO对象存储的使用，理解了分布式文件存储的优势和应用场景。
5. 通过系统集成测试，理解了前后端协作的重要性，学会了使用接口文档进行联调。

**存在不足**：
1. 数据库设计初期考虑不够全面，后期频繁修改表结构，导致需要编写多个数据库迁移脚本。
2. 二级评论的递归查询效率不高，当评论数量较多时响应较慢，可以使用缓存或非关系型数据库优化。
3. 测试用例设计不够全面，部分边界条件没有覆盖到，导致上线后出现了一些问题。
4. 文档编写不够及时，部分接口没有更新到接口文档中，导致前后端联调时出现误解。

**改进方向**：
1. 在项目初期进行充分的数据库设计评审，减少后期修改，使用数据库版本管理工具（如Flyway）管理变更。
2. 优化评论查询性能，可以将评论树缓存到Redis中，或者使用MongoDB等非关系型数据库存储评论数据。
3. 加强测试用例设计，使用等价类划分、边界值分析等方法设计测试用例，提高测试覆盖率。
4. 及时更新接口文档，使用Swagger等工具自动生成API文档，确保前后端信息同步。
5. 学习CI/CD流程，使用Jenkins或GitHub Actions实现自动化测试和部署。

## 团队协作总结

### 协作模式

本项目采用**前后端分离+敏捷开发**的协作模式：

1. **需求分析阶段**：三人共同讨论项目需求，确定功能模块和技术选型
2. **设计阶段**：
   - 成员一负责后端API设计（接口定义、请求响应格式）
   - 成员二负责前端页面设计（页面布局、交互流程）
   - 成员三负责数据库设计（表结构、ER图）
3. **开发阶段**：
   - 采用Git进行版本控制，建立develop、feature、master分支
   - 每日站会同步进度，及时解决阻塞问题
   - 使用Mock数据并行开发前后端
4. **联调测试阶段**：
   - 成员三负责组织前后端联调
   - 使用Postman进行接口测试
   - 发现问题及时分配给对应负责人修复

### 版本管理

```
master        ───────────────────────────────────  生产环境
               ↓
develop       ───────────────────────────────────  开发环境
     ↓         ↓         ↓         ↓
feature/auth  feature/q  feature/a  feature/c  (功能分支)
```

### 遇到的问题与解决方案

| 问题 | 解决方案 |
|------|---------|
| 前后端接口定义不一致 | 使用Swagger生成接口文档，约定请求响应格式 |
| 数据库表结构频繁变更 | 使用Flyway管理数据库版本，编写迁移脚本 |
| Git合并冲突频繁 | 合理划分功能模块，减少同一文件的并发修改 |
| 联调时环境配置不一致 | 统一开发环境配置，编写环境变量设置指南 |
| 代码风格不统一 | 使用ESLint、Prettier统一代码格式，配置pre-commit钩子 |

### 项目收获

通过本次项目，三人团队不仅掌握了Java Web开发的全栈技术，更重要的是学会了团队协作：

1. **沟通能力**：通过每日站会、代码评审等方式保持信息同步
2. **分工协作**：根据个人特长合理分配任务，发挥各自优势
3. **问题解决**：遇到技术难题时共同讨论、互相帮助
4. **项目管理**：学会了使用Git、Jenkins等工具进行项目管理和版本控制
